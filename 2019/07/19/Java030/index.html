异常的捕获及处理

异常是导致程序中断执行的一种指令流，异常一旦出现并且没有进行合理处理的话，那么程序就将中断执行。

处理异常

Java提供了三个关键字： try、catch、finally，语法格式：

对于以上的操作组合：try…catch、try…catch…finally、try…finally（不建议使用）。
可以使用e.printStackTrace()函数输出异常完整信息。

异常处理流程（核心）

可以使用 Exception 统一处理异常，所以在程序中不用关心到底使用哪一个异常。有两点说明：

  • 在编写捕获异常的时候，捕获范围大的异常一定要放在捕获范围小的异常之后，否则编译错误。
  • 虽然直接捕获 Exception 比较方便，但是也不好，因为所有的异常都会按照同样一种方式处理。如果在一些要求严格的项目里，异常一定要分开处理会更好。

throws关键字

throws是用于方法声明上，指的是当方法之中出现异常后交由被调用处来处理。
调用了具有throws声明方法之后，那么不管操作是否出现异常，都必须使用try…catch来进行异常的处理。
如果在主方法上继续抛出异常，那么这个异常就交给JVM处理，而后结束程序调用。
主方法上不要加throws，因为程序及时出错也希望可以正常的结束调用。

throw关键字

程序中可以使用throw手工抛出一个异常类的实例化对象。
throws和throws的区别:

  • throw：指的是在方法中人为抛出一个异常类对象（这个异常类对象可能是自己实例化或者是抛出已存在的）
  • throws：在方法的声明上使用，表示此方法在调用时必须处理异常。

异常的使用格式

RuntimeException类

RuntimeException类，程序在编译时不会强制性的要求用户处理异常，用户可以根据自己的需要选择性进行处理，但是如果没有处理又发生了，将交给JVM默认处理。也就是说RuntimeException的子异常类，可以由用户根据需要选择进行处理。

Exception 和 RuntimeExceptiond 的区别

  • Exception 是 RuntimeException 的父类
  • 使用 Exception 定义的异常必须要被处理，而 RuntimeException 的异常可以选择性处理

常见的 RuntimeException：ArithmeticException、NullPointerException、ClassCastException。

assert关键（了解）

assert主要功能是进行断言。
Java中的断言指的是程序执行到某处代码时，得到的一定是预期的结果。
默认情况下断言是不应该影响程序的运行的，也就是说在Java解释程序的时候默认是不起作用的。
启用断言：java -ea TestDemo（TestDemo是文件名）

自定义异常

Java本身已经提供了大量的异常，但是往往实际工作中不起作用。
选择自定义异常可以选择继承 Exception 或 RuntimeException。

总结

  1. Exception 的父类是 Throwable，但是在编写代码的时候尽量不要使用。因为 Throwable 下还有一个 Error 子类，我们能够处理的只有 Exception 子类。
  2. 异常处理的标准格式：try、catch、finally、throw、throws一起使用
  3. RuntimeException 和 Exception 的区别

类图

使用 PowerDesigner 可以描述出类的结构图与时序图。

类图的组成结构：

  • 第一层：类名称，如果是抽象类使用斜体字
  • 第二层：描述类中的属性，对于属性如果要封装使用“-”表示。public（+）、protected（#）、private（-）
  • 第三层：类中定义的方法

时序图是描述整个代码的执行步骤。

总结

对于 UML 的类图基本上都是自动转换，而利用时序图可以很好的描述出程序的执行逻辑。
